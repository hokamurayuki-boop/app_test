# 夜勤手当計算アプリ 計画書

作成日: 2026-04-11

---

## 1. 目的

エクセルで管理している勤怠データから、規定に基づく夜勤手当を自動計算し、結果をエクセルへ書き戻す（または別シートに出力する）アプリを作成する。

手計算・目視による集計の手間とミスをなくし、給与計算業務を効率化することを目的とする。

---

## 2. 対象業務（現状）

- 勤怠は Excel ブックで管理されている。
- 給与計算ソフトの仕様上、**同一シフトでも日付が変わるとエクセル行が分割される**。
- 夜勤手当は、勤怠担当者が目視で各行を確認しながら計算している（と想定）。

---

## 3. 計算ロジック（要件）

### 3.1 基本ルール
| 項目 | 内容 |
|---|---|
| 夜勤時間帯 | **22:00 〜 翌 08:00** |
| 単価 | **1,000 円 / 時間**（夜勤時間帯にかかった分のみ） |
| 1勤務あたりの上限 | **5,000 円** |

### 3.2 行をまたぐシフトの扱い（重要）

給与計算ソフトは日付をまたぐと行を分割するため、**1つのシフトが2行に分かれて記録される**。
アプリは以下の手順で処理する必要がある。

1. **シフトのグルーピング**
   - 同一従業員 × 連続する時間帯（前行の終了時刻 = 次行の開始時刻）を 1 シフトとしてまとめる。

2. **行ごとの夜勤時間算出**
   - 各行の勤務時間と夜勤時間帯（22:00〜翌8:00）の重なりを分単位で計算。

3. **シフト合計で上限判定**
   - シフト単位で合計夜勤手当（仮）を算出。
   - 5,000 円を超える場合は **5,000 円に丸める**。

4. **行への再配分**
   - 先頭行から順に割り当て、上限に達したら以降の行を切り詰める（FIFO 方式）。
   - 例示のケースと一致することを確認済み。

### 3.3 具体例（仕様書の例）

勤務: 4/10 20:00 〜 4/11 08:00

| 行 | 開始 | 終了 | 行内勤務時間 | 夜勤時間(22-8) | 素点手当 | 配分後手当 |
|---|---|---|---|---|---|---|
| 1 | 4/10 20:00 | 4/11 00:00 | 4h | 2h (22-24) | 2,000 | **2,000** |
| 2 | 4/11 00:00 | 4/11 08:00 | 8h | 8h (0-8) | 8,000 | **3,000** |
| 計 | | | 12h | 10h | 10,000 | **5,000** |

→ シフト合計 5,000 円上限。Row1 に 2,000 割当 → Row2 に残り 3,000 割当。

## 4. 確認したい事項（要ヒアリング）

計画確定のため、以下を確認させてください。

1. **エクセルの構造**
   - ブック / シート名・行列の構成（列名、ヘッダ行の位置）
   - 従業員を識別する列（社員番号？ 氏名？）
   - 日付・開始時刻・終了時刻の列の形式（日付と時刻が同じセル？ 別セル？）

2. **シフト境界の判定方法**
   - 同一シフト判定は「前行の終了 == 次行の開始」で厳密に連結するか？
   - それとも「同日 or 翌日0:00で分割」というシンプルな前提でよいか？

3. **出力方法**
   - 既存シートに「夜勤手当」列を追加して書き込みか
   - 別シート / 別ファイルに集計出力か
   - 従業員ごとの月次合計も必要か

4. **上限配分の方式**
   - 例の通り「先頭行優先（FIFO）」でよいか
   - あるいは「時間数按分」等の別方式があるか

5. **エクセルファイルの取り扱い**
   - ローカル PC 上のファイルを開いて処理するか
   - OneDrive / SharePoint 上か
   - 実行中に他ユーザーが開いている可能性はあるか

6. **既存スキル「night-shift-allowance」との関係**
   - 同名の社内スキル（訪問介護シフト CSV → Google Spreadsheet 転記）が存在するが、今回は **Excel ⇄ Excel** のため別実装になる想定。
   - 計算ロジックのみ流用可能か確認する。

## 5. システム構成案

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│ 勤怠 Excel ファイル │ →  │ 夜勤手当計算アプリ   │ →  │ 結果出力         │
│ (.xlsx)         │    │ (Python / openpyxl)│    │ (同ファイル or別) │
└─────────────────┘    └──────────────────┘    └─────────────────┘
                              │
                              ├─ ①読み込み・パース
                              ├─ ②シフト単位グルーピング
                              ├─ ③夜勤時間計算
                              ├─ ④上限適用
                              └─ ⑤行へ再配分・書き戻し
```

### 5.1 技術選定（案）

| 候補 | メリット | デメリット |
|---|---|---|
| **Python + openpyxl**（推奨） | 無料・軽量・保守しやすい。書式を保持して読み書き可能 | 実行環境に Python が必要 |
| Excel VBA | Excel 単体で完結、配布が楽 | 保守しづらい・テスト自動化が難しい |
| Power Query / Power Automate | ノーコードに近い | 条件付き再配分ロジックが複雑で表現が難しい |

現時点では **Python + openpyxl** を推奨。CLI または簡易 GUI（tkinter）で提供可能。

## 6. データモデル（内部表現）

```python
@dataclass
class AttendanceRow:
    row_index: int          # Excel 上の行番号
    employee_id: str
    start: datetime         # 開始日時
    end: datetime           # 終了日時
    raw_night_minutes: int  # 夜勤帯の重なり（計算後）
    raw_allowance: int      # 素点手当（計算後）
    final_allowance: int    # 上限配分後の手当（出力値）

@dataclass
class Shift:
    employee_id: str
    rows: list[AttendanceRow]  # 連続する行のリスト
    total_raw: int
    total_final: int           # min(total_raw, 5000)
```

## 7. 実装ステップ

1. **サンプルデータの取得**（実ファイル 1 週間分程度）
2. **読み込み処理**: 従業員・日時列をパースして `AttendanceRow` 化
3. **夜勤時間算出関数**: `(start, end) → 22-8 帯と重なる分数`（単体テスト必須）
4. **シフトグルーピング**: 従業員単位で時系列ソート→連結判定
5. **上限適用＋FIFO 再配分**
6. **Excel 書き戻し**（指定列へ数値を書き込み、書式を崩さない）
7. **テスト**: 仕様書の例・上限未満・上限超過・単独行・3行以上のシフト など
8. **運用**: 実行手順書の作成、担当者への引き渡し

## 9. リスク・懸念点

- **シフト連結の誤判定**: わずかな時刻ずれで別シフト扱いになる恐れ → 許容誤差（例: 0分厳密 or ±5分）をヒアリング。
- **Excel 書式破壊**: openpyxl は一部書式を保持できないケースあり → 実ファイルで事前検証。
- **同時編集**: 担当者が Excel を開いていると書き込めない → 実行前のクローズを運用ルールに。
- **休憩時間の扱い**: 夜勤帯内に休憩がある場合、控除するか不明 → 要確認。
