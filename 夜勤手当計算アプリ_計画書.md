## 0. アーキテクチャ方針（2026-04-11 更新）

**Python + openpyxl ローカル実行案 → Netlify 静的Webアプリ案** に変更。

| 項目 | 採用内容 |
|---|---|
| 公開方法 | Netlify (静的ホスティング) |
| 実装 | HTML + JavaScript (ES5/ES6) |
| Excel読み書き | [SheetJS (xlsx)](https://sheetjs.com/) を CDN から利用 |
| サーバー処理 | **なし**（勤怠データは全てブラウザ内で処理） |
| 運用 | ブラウザでURLにアクセス → ファイル選択 → 計算 → 結果ダウンロード |

**採用理由:**
- 勤怠データ（個人情報）がサーバーに送信されないため、情報セキュリティ観点で安全。
- Python実行環境の構築・配布が不要。担当者は URL を開くだけで使える。
- Netlify 無料枠で運用可能。デプロイは Git push で自動。
- 計算ロジックは当初の Python 案と同一（計画書 3章）なので、要件は全く変わらない。

**ファイル構成:**
```
アプリテスト/
├── index.html       # UI
├── app.js           # 計算ロジック + Excel I/O + テスト
├── style.css        # スタイル
├── netlify.toml     # Netlify 設定
└── 夜勤手当計算アプリ_計画書.md
```

## 1. 目的

エクセルで管理している勤怠データから、規定に基づく夜勤手当を自動計算し、結果をエクセルへ書き戻す（または別シートに出力する）アプリを作成する。

手計算・目視による集計の手間とミスをなくし、給与計算業務を効率化することを目的とする。

---

## 2. 対象業務（現状）

- 勤怠は Excel ブックで管理されている。
- 給与計算ソフトの仕様上、**同一シフトでも日付が変わるとエクセル行が分割される**。
- 夜勤手当は、勤怠担当者が目視で各行を確認しながら計算している（と想定）。

---

## 3. 計算ロジック（要件）

### 3.1 基本ルール（確定 2026-04-11）
| 項目 | 内容 |
|---|---|
| 夜勤時間帯 | **22:00 〜 翌 08:00** |
| 単価 | **500 円 / 30分ブロック**（＝実効 1,000円/時間） |
| 丸め | **30分未満は切り上げ**: `row_raw = ceil(night_min / 30) × 500` |
| 1勤務あたりの上限 | **5,000 円** |

**切り上げの例:**
- 10分 → ceil(10/30)=1 → 500円
- 30分 → ceil(30/30)=1 → 500円
- 40分 → ceil(40/30)=2 → 1000円
- 60分 → ceil(60/30)=2 → 1000円
- 70分 → ceil(70/30)=3 → 1500円

### 3.2 行をまたぐシフトの扱い（重要）

給与計算ソフトは日付をまたぐと行を分割するため、**1つのシフトが2行に分かれて記録される**。
アプリは以下の手順で処理する必要がある。

1. **シフトのグルーピング**
   - 同一従業員 × 連続する時間帯（前行の終了時刻 = 次行の開始時刻）を 1 シフトとしてまとめる。

2. **行ごとの夜勤時間算出**
   - 各行の勤務時間と夜勤時間帯（22:00〜翌8:00）の重なりを分単位で計算。

3. **シフト合計で上限判定**
   - シフト単位で合計夜勤手当（仮）を算出。
   - 5,000 円を超える場合は **5,000 円に丸める**。

4. **行への再配分**
   - 先頭行から順に割り当て、上限に達したら以降の行を切り詰める（FIFO 方式）。
   - 例示のケースと一致することを確認済み。

### 3.3 具体例

勤務: 4/10 20:00 〜 4/11 07:00（CSV上は2行に分割されている）

| 行 | 開始 | 終了 | 夜勤分数 | 素点(切り上げ) | 配分後 |
|---|---|---|---|---|---|
| 1 | 4/10 20:00 | 4/11 00:00 | 120分 | `ceil(120/30)*500` = 2,000 | **2,000** |
| 2 | 4/11 00:00 | 4/11 07:00 | 420分 | `ceil(420/30)*500` = 7,000 | **3,000** |
| 計 | | | 540分 | 9,000 → cap 5,000 | **5,000** |

→ シフト合計 5,000 円上限。Row1 に 2,000 割当 → Row2 に残り 3,000 割当（FIFO）。

## 4. 入力CSV仕様（確定）

| 項目 | 内容 |
|---|---|
| ファイル形式 | **CSV (Shift-JIS)** |
| 列数 | 10列固定 |
| ヘッダー | 1行目 |
| 列 | ヘルパー名 / 日付 / 曜日 / 利用者 / 業務種別 / サービス内容 / 開始時間 / 終了時間 / 提供時間（分） / 備考 |
| 日付 | **「日」のみ**（例: "3"）。年月はファイル名から自動抽出 |
| 時刻 | `HH:MM` 形式。`24:00` は翌日 00:00 として扱う |

### 4.1 年月の抽出ルール（ファイル名から）
優先順: 令和 → 平成 → R8 → 2026年4月 → 2026-04 → 2026_04 → 2026/4 → 202604

ファイル名から抽出できない場合、UIで手動入力。

| 項目 | 採用 | 理由 |
|---|---|---|
| ランタイム | ブラウザ JavaScript (ES6) | インストール不要、データ外部送信なし |
| CSV ライブラリ | 自前実装（簡易パーサー） | 外部依存ゼロ、10列固定で十分 |
| エンコーディング | `TextDecoder('shift_jis')` | 標準API |
| ホスティング | Netlify | Git 連携で自動デプロイ、無料枠で十分 |
| ビルドツール | なし | 静的 HTML/JS のみ、シンプル |

## 8. リスク・懸念点

- **Shift-JIS デコード失敗**: UTF-8 の新形式が来た場合はフォールバック済み。
- **シフト連結の誤判定**: ±1分の許容あり。それを超えるズレは別シフト扱い。
- **ファイル名に年月がない**: UI で手入力するフォールバックあり。
- **休憩時間の扱い**: 夜勤帯内に休憩がある場合は CSV 側で別行になっている前提（現状 CSV にはその表現なし）。

4. Build settings: デフォルトのまま（`netlify.toml` の設定が適用される）
5. Deploy site → 数秒で公開URLが発行される
